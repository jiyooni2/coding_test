
Greedy
    탐욕적으로 문제를 푸는 알고리즘
    현재 상황에서 지금 당장 좋은 것만 고르는 방법
    매 순간 가장 좋아 보이는 것을 선택, 현재의 선택이 나중에 미칠 영향에 대해 고려 x
    정렬 알고리즘과 자주 짝을 이뤄 출제
    문제를 풀기 위한 최소한의 아이디어를 떠올릴 수 있는 능력 요구
    해법이 정당한지 검토 필요

Implementation
    머릿속에 있는 알고리즘을 소스코드로 바꾸는 과정
    
    완전 탐색
        모든 경우의 수를 주저 없이 다 계산하는 해결 방법
    
    시뮬레이션
        문제에서 제시한 알고리즘을 한 단계식 차례대로 수행

    1초 = 2000만번

DFS
    스택 사용 or 재귀 함수 사용
    1. 탐색 시작 노드를 스택에 삽입하고 방문 처리
    2. 방문 X 인접 노드 스택에 넣고 방문 처리, 없으면 스택에서 pop
    3. 반복

    활용 : 완전 탐색, 갈 수 있는 모든 곳

BFS
    큐 사용
    1. 큐에 삽입 후 방문 처리
    2. 큐에서 노드 꺼낸 뒤 방문 X 인접 노드 모두 큐에 삽입 후 방문 처리
    3. 반복

    활용 : 최단거리, 몇 번 안에 도달 가능한 곳 모임
